ローカル確認
index.html をブラウザで開いてください。

ネットワーク化
1. Googleスプレッドシートを新規作成
2. 拡張機能 → Apps Script
3. google_apps_script_backend.gs を貼り付け
4. API_KEY を自分だけの文字列に変更
5. デプロイ → 新しいデプロイ → ウェブアプリ
6. 実行ユーザー：自分
7. アクセスできるユーザー：自分のみ推奨
8. 発行されたWebアプリURLをPWAの初期設定へ貼り付け
9. アクセスキーも同じものを入力
10. 同期ボタンでスマホ・PC間同期

公開先
GitHub Pages / Netlify / Cloudflare Pages などにアップロードできます。

注意
患者氏名と通院履歴は個人情報です。
Apps Scriptのアクセス範囲は「自分のみ」、端末ロック、バックアップJSONの管理を推奨します。


修正版メモ
・患者No.数字入力判定の不具合を修正しました。
・日付欄を直接入力ではなく、年/月/日をスクロール選択できる形式に変更しました。


同期安全版 修正内容
・同期時にGoogle側が空の場合、ローカルの保存済みデータを優先してアップロードします。
・ローカルにデータがあり、クラウド側が空のときに空データで上書きされないようにしました。
・同期失敗時は同期前のローカルデータに戻します。
・同期前後はバックアップJSONの保存を推奨します。

重要
このZIPは「別URLテスト用」です。
本番URLへ上書きする前に、現在のアプリでバックアップJSONを保存してください。

LINE通知について
ブラウザ側にLINEトークンを置かない設計です。
Apps Script側の以下を設定してください。
・LINE_CHANNEL_ACCESS_TOKEN
・LINE_TO_ID

freeeについて
現時点ではfreee取込用CSVの自動作成です。
freee APIへの完全自動送信にはfreee側のOAuthアプリ登録が必要です。
